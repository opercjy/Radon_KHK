
# Korean Group - Integrated DAQ & Environment Monitor

FTLAB RS-9A 라돈(Radon) 센서의 실시간 데이터를 수집(DAQ) 및 시각화하고, 연속 측정 데이터를 엑셀(Excel) 로그 파일로 안전하게 기록하기 위한 Windows 기반 통합 모니터링 애플리케이션입니다.

---

## 📋 폴더 구조 (Project Structure)
```text
Radon_KHK/
│
├── main.py                 # 프로그램 실행 진입점 (Main Entry)
├── config.json             # 사용자 환경설정 파일 (소속 기관 및 작업자 명칭)
│
├── core/                   # 데이터 처리 및 하드웨어 통신 백그라운드 모듈
│   ├── daq_poller.py       # 광학 센서(PMT/SiPM) 파형 시뮬레이션 제어 스레드 (임시)
│   └── rs9a_poller.py      # RS-9A 시리얼 통신 제어 스레드 (버퍼 딜레이 방지 로직 적용)
│
└── ui/                     # 사용자 인터페이스(GUI) 및 화면 제어 모듈
    ├── main_window.py      # 윈도우 메인 프레임 및 탭 레이아웃 관리
    ├── tab_daq.py          # 광학 센서 제어 패널 UI
    └── tab_radon.py        # RS-9A 센서 제어, 실시간 렌더링 및 엑셀 추출 UI

```

## 🛠️ 1. 하드웨어 드라이버 설치 (Silicon Labs CP210x) - **[필수]**

RS-9A 센서는 UART 시리얼 통신 방식을 사용합니다. 센서를 PC의 USB 단자에 연결했을 때 Windows가 정상적인 COM 포트로 인식하게 하려면, 통신 변환 칩셋인 **Silicon Labs CP210x 드라이버**를 반드시 수동으로 설치해야 합니다.

### [ 📥 윈도우 환경 상세 설치 가이드 ]

1. **드라이버 다운로드 페이지 접속:**
* [Silicon Labs 공식 다운로드 페이지 (클릭)](https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers)로 이동합니다.


2. **드라이버 파일 다운로드:**
* 웹페이지 중간의 `Downloads` 탭을 클릭합니다.
* **`CP210x Universal Windows Driver`** (Windows 10/11 전용) 항목을 클릭하여 `.zip` 압축 파일을 다운로드합니다.


3. **압축 해제 및 설치:**
* 다운로드한 파일의 압축을 바탕화면 등 찾기 쉬운 경로에 풉니다.
* 압축을 푼 폴더 내부에서 톱니바퀴가 그려진 문서 모양 아이콘인 **`silabser.inf`** (Setup Information / 설치 정보) 파일을 찾습니다. *(주의: silabser.sys 나 .cat 파일이 아닙니다.)*
* 해당 파일 위에 **마우스 우클릭 ➔ [설치(I)]** 를 클릭합니다.
* *"작업을 완료했습니다"* 라는 윈도우 알림창이 뜨면 드라이버가 시스템에 성공적으로 등록된 것입니다.


4. **포트 인식 확인 및 COM 번호 메모:**
* RS-9A 라돈 센서를 PC의 USB 포트에 연결합니다.
* 키보드의 `Windows 키 + X` 를 누르고 [장치 관리자(Device Manager)]를 실행합니다.
* **[포트(COM & LPT)]** 항목을 확장합니다.
* 목록에 **`Silicon Labs CP210x USB to UART Bridge (COM3)`** 형식으로 기재되어 있는지 확인하고, 할당된 `COM 번호(예: COM3, COM4)`를 기억해 둡니다.

## 💻 2. 소프트웨어 개발 환경 세팅 (파이썬)

소스 코드를 직접 구동하려면 파이썬(Python 3.8 이상)과 필수 라이브러리가 필요합니다. 파이썬이 설치된 상태에서 명령 프롬프트(CMD) 또는 PowerShell을 열고, 프로젝트 폴더 경로로 이동한 뒤 아래 명령어를 차례대로 입력하세요.

```bash
# 기본 데이터 연산 및 시리얼 통신, 엑셀 처리 라이브러리 설치
pip install numpy pandas openpyxl pyserial

# GUI 화면 렌더링 및 고속 그래프 시각화 라이브러리 설치
pip install PyQt6 pyqtgraph

```

## 🚀 3. 프로그램 실행 및 사용 방법

터미널(CMD)의 프로젝트 최상위 폴더에서 메인 파일을 실행합니다.

```bash
python main.py

```

### 💡 주요 작동 가이드

1. **포트 스캔:** 첫 화면 콤보박스 옆의 `[Scan]` 버튼을 눌러 장치 관리자에서 확인했던 `COM 포트(Silicon Labs...)`를 찾아 선택합니다.
2. **측정 시작:** `[Start]` 버튼을 눌러 통신을 개시합니다.
3. **데이터 추출:** 모니터링 중이거나 측정을 종료한 후, 언제든지 `[Export Excel]` 버튼을 눌러 수집된 1분 단위 연속 데이터를 `.xlsx` 엑셀 파일로 저장할 수 있습니다.

> **⚠️ 물리적 대기 시간 안내 (Troubleshooting)**
> * **초기화 대기:** Start를 눌러도 즉각적인 데이터가 나오지 않고 **`Initializing...`** 상태로 머무릅니다. 이는 펄스 전리함 센서 내부의 알파(α) 붕괴 카운팅 누적을 위한 정상적인 물리적 대기 시간(약 10~30분)입니다. 고장이 아니며, 이 대기 구간 동안에도 엑셀 로그에는 생존 통신 기록이 정상적으로 남습니다.
> * **수치 정체 현상:** 시간 단위의 이동평균(Moving Average) 필터를 거친 농도를 출력하므로, 환경 변화가 크지 않으면 소수점까지 동일한 수치가 오랜 시간 유지될 수 있습니다. 하단 그래프의 X축(시간)이 1분 간격으로 끊김 없이 오른쪽으로 그려지고 있다면 시스템은 완벽히 정상 동작 중입니다.

## 📦 4. 현장 배포용 실행 파일(.exe) 빌드 가이드

파이썬 환경 구축이 어려운 현장 PC나 일반 작업자용 컴퓨터에서 클릭 한 번으로 실행 가능하도록 독립 실행 파일(.exe) 환경을 구축하는 방법입니다.

**1. 빌드 도구 설치:**

```bash
pip install pyinstaller

```

**2. 폴더 형태 빌드 실행 (콘솔 창 숨김 옵션 적용):**
프로젝트 최상단 폴더에서 아래 명령어를 실행합니다.

```bash
pyinstaller --noconsole --onedir main.py

```

**3. 배포 및 실행 방법:**

* 빌드가 완료되면 프로젝트 내부에 `dist`라는 새 폴더가 생성되고, 그 안에 **`main`** 이라는 디렉토리가 만들어집니다.
* **[주의사항]** 빌드 과정에서 환경설정 파일은 복사되지 않습니다. 프로젝트 원본 폴더에 있는 **`config.json` 파일을 `dist/main/` 폴더 내부(`main.exe`가 위치한 곳)로 직접 복사**하여 붙여넣으세요.
* 완료 후 `dist/main` 폴더 전체를 USB 등에 담아 현장 PC로 복사한 뒤, **`main.exe`** 파일만 더블클릭하면 셋업 없이 즉시 프로그램을 사용할 수 있습니다.
